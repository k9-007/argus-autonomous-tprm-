* About the project

Paste the Markdown below into Devpost (it accepts Markdown). A rendered preview follows.

Inspiration

Every company now runs on dozens of SaaS and AI vendors — and before adopting each one, someone has to vet its security posture: collect SOC 2 / ISO reports, read the DPA, check for breaches, map controls, fill a 100–300 question security questionnaire, then keep watching for changes. At mid-market and AI-first companies there is no GRC analyst, so it lands on an engineer buried in spreadsheets. Reviews take weeks, block deals, and go stale the day they finish. AI tools, agents and MCP servers make it worse — the fastest-growing, riskiest vendor class, with no traditional playbook.

What it does

Add a vendor via upload or trust-center link, and a crew of nine AI agents autonomously profiles & tiers it, ingests gated evidence, maps controls across SOC 2 / ISO 27001 / GDPR / HIPAA / PCI / NIST / ISO 42001, auto-completes SIG/CAIQ with citations, scores explainable residual risk, routes NDAs to a human Approver, arms continuous monitoring, and writes a board-ready decision — all live in a portfolio + per-vendor dashboard with a real-time risk graph and a cross-org Trust Passport network.

How we built it

Python + FastAPI + SQLAlchemy backend; Next.js + React + TypeScript dashboard; model-flexible reasoning (OpenAI GPT-5.6 or Google Gemini) with structured outputs and offline deterministic fallbacks; Bright Data discovery tools; a real PDF/text evidence parser. Designed and accelerated with OpenAI Codex + GPT-5.6.

Challenges · Accomplishments · What we learned · What's next

See the full Markdown above — it covers the trust-center NDA reality, explainable scoring, honest evidence parsing, offline reliability, first-class AI-vendor risk, the Trust Passport network effect, and the roadmap.

Installation & testing instructions

Only required for plugins / dev tools. Argus is a web app, so this is optional — provided for judges who want to run it.
Not a plugin/dev tool — Argus is a web app (FastAPI backend + Next.js frontend). It runs fully offline with deterministic heuristics, so judges can test it with no API keys. Supported platforms: any modern browser (frontend); macOS / Linux / Windows for the backend. Prerequisites: Python 3.12+ and Node 18+. --- Backend (port 8000) --- cd backend python3 -m venv .venv && source .venv/bin/activate pip install -r requirements.txt cp .env.example .env # optional: set ARGUS_LLM_PROVIDER + OPENAI_API_KEY or GEMINI_API_KEY uvicorn app.main:app --port 8000 # health: http://127.0.0.1:8000/health · docs: http://127.0.0.1:8000/docs --- Frontend (port 3000) --- cd frontend npm install cp .env.local.example .env.local npm run dev # open http://localhost:3000 --- How to test --- - Sign up, or click "Use demo account" (demo@acme.com / demo1234). - Add "Cursor" via trust-center link, "Stripe" via upload, and "Acme MCP" for AI-risk. - Watch the live 9-agent crew flow, then explore Compliance, AI Risk, and the real-time Monitoring graph. - Optional live reasoning: set ARGUS_LLM_PROVIDER=openai + OPENAI_API_KEY (GPT-5.6) or ARGUS_LLM_PROVIDER=gemini + GEMINI_API_KEY.
